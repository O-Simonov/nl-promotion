# Цена и характеристики товара NL по названию

Скилл достаёт с витрины интернет-магазина NL International **актуальную цену, наличие И характеристики** любого товара (фасовка, КБЖУ, ключевые свойства, артикул, награды) через Playwright MCP (живой браузер). Данные динамические — берём их с сайта, а НЕ из памяти.

Это единый источник правды о товаре для всего проекта: и для ответа «сколько стоит», и для наполнения постов реальными данными (см. скилл `nl-autopost`).

## Когда применять

- «Зайди на сайт NL и проверь цену <товар>»
- «Сколько сейчас стоит / характеристики / состав / КБЖУ <товар>»
- Нужно наличие («есть / нет в наличии»), фасовка, артикул
- **Перед созданием поста про конкретный продукт** — взять цену и характеристики с сайта
- Нужно собрать данные сразу по нескольким товарам или по линейке

## Предусловие

Playwright MCP-сервер уже установлен в проекте (браузер для чтения динамических страниц NL). Если инструменты `mcp__playwright__browser_*` недоступны — сначала загрузить их схемы через ToolSearch (`select:mcp__playwright__browser_navigate,...`).

## Ключевые факты о сайте (проверено)

1. **Магазин — это SPA на `ng.nlstar.com`**, НЕ `nlstar.com`.
   `nlstar.com` — корпоративный сайт, его `/search/` отдаёт **404**.
2. **Рабочий поиск:** `https://ng.nlstar.com/ru/search?q=<запрос>`
   (параметр именно `q`; при ручном вводе в поле поиска URL сам станет `?q=...`).
3. **Контент грузится асинхронно.** Сразу после `navigate` блок `main` пустой — обязательно `browser_wait_for` 2–3 секунды, затем `browser_snapshot`.
4. **Цена и наличие зависят от региона** (по умолчанию Москва). Для другого города — сменить регион кнопкой «Москва | ru» в шапке.
5. **Бренд ≠ один товар.** «Greenflash», «Energy Diet» и т.п. — это линейки из десятков позиций. Поиск выдаёт их вперемешку с сетами; нужно выбрать конкретный товар.
6. **Сет (набор) ≠ одиночный товар.** Сеты (`/ru/set/...`) стоят кратно дороже одиночных товаров (`/ru/product/...`) — не путать.

## Процедура

### Шаг 1 — Открыть поиск магазина
```
mcp__playwright__browser_navigate  url: https://ng.nlstar.com/ru/search?q=<URL-encoded запрос>
```
Пробел кодировать как `%20` или `+`. Пример: `?q=Energy%20Diet`.

### Шаг 2 — Дождаться отрисовки и снять снапшот
```
mcp__playwright__browser_wait_for   time: 3
mcp__playwright__browser_snapshot
```
Cookie-баннер в шапке чтению не мешает — игнорировать (или нажать «OK», если мешает клику).

> Запасной путь, если страница поиска не отрисовала результаты: ввести запрос в поле поиска —
> `browser_type` в `searchbox "Искать в NL Store…"` с `submit: true`, потом снова `wait_for` + `snapshot`.

### Шаг 3 — Найти товар в результатах
Раздел `heading "Продукция"` содержит карточки: название (ссылка `/ru/product/<id>` или `/ru/set/<slug>`), цена `<сумма> ₽`, статус («В корзину» / «Выбрать» / «Нет в наличии»). Выбрать нужную карточку по названию.

### Шаг 4 — Открыть карточку и снять ВСЕ данные о товаре
```
mcp__playwright__browser_navigate  url: https://ng.nlstar.com/ru/product/<id>
mcp__playwright__browser_wait_for  time: 2
mcp__playwright__browser_take_screenshot   (нагляднее всего; либо browser_snapshot)
```
**С карточки забирать (это и есть «характеристики с сайта»):**

| Поле | Где на карточке |
|------|-----------------|
| Точное название | заголовок товара |
| **Цена** | первое `<сумма> ₽` (остальные суммы ниже — блок «рекомендуем», игнорировать) |
| Фасовка | «N порций по M г» / объём / кол-во штук |
| Артикул | «Арт. NNNNN» |
| Наличие | «В корзину» = есть; «Нет в наличии» = нет |
| КБЖУ | «КБЖУ на 1 порцию…»: ккал, белки, жиры, углеводы, волокна |
| Ключевые свойства | абзац описания + буллеты («можно на воде и на молоке» и т.п.) |
| Награды / статус | «Марка №1 в России», «Хит», «Новинка» |
| Состав (если нужен) | вкладка «Состав» — нажать `browser_click`, затем снапшот |

### Шаг 5 — Закрыть браузер
```
mcp__playwright__browser_close
```

## Чтение данных из большого снапшота

Снапшот карточки крупный и сохраняется в файл. Чтобы вытащить только цены:
```
Grep  pattern: \d[\d  ]*₽   output_mode: content   -o: true   path: <путь к сохранённому снапшоту>
```
**Первая** найденная сумма — цена товара. Для КБЖУ/фасовки удобнее `browser_take_screenshot` и прочитать визуально.

## Формат ответа пользователю (цена)

```
Актуальная цена (NL Store, регион <город>, <дата>):

Товар:     <точное название>
Цена:      <сумма> ₽
Фасовка:   <N порций по M г / объём>
Артикул:   <id>
Наличие:   в наличии / нет в наличии
```
Несколько товаров (вкусы, объёмы, сеты) → короткая таблица; пометить, что цена единая за вкус, а сеты считаются отдельно.

## Формат данных для поста (цена + характеристики)

Собрать «карточку факта» и передать в `nl-autopost`:
```
Название:    <точное>
Цена:        <сумма> ₽   (+ цена за порцию, если есть фасовка)
Фасовка:     <N порций по M г / объём>
КБЖУ/порция: <ккал, белки, жиры, углеводы, волокна>   (для питания)
Свойства:    <2–4 реальных преимущества из описания>
Награды:     <если есть>
Наличие:     <есть / нет>
Артикул:     <id>
URL:         https://ng.nlstar.com/ru/product/<id>
```
⚠️ В пост попадают только реально прочитанные с сайта значения. Ничего не выдумывать и не брать из старых постов — цены и фасовки на сайте меняются (например, ED Smart: было «14 порций / 2 796 ₽», стало «15 порций / 2 790 ₽»).

## Диагностика

| Симптом | Причина | Решение |
|---------|---------|---------|
| 404 / «Что-то пошло не так» | Открыт `nlstar.com` вместо магазина | Использовать `ng.nlstar.com` |
| `main` пустой в снапшоте | SPA не успела отрисоваться | `browser_wait_for` 3 сек и снять снапшот заново |
| «Target page … has been closed» | Гонка между navigate и snapshot | Повторить `navigate`, затем `wait_for`, затем `snapshot` |
| Цена/характеристики не те | Другой регион / выбран «сет», а не товар | Сменить регион; открыть именно `/ru/product/<id>` |
| Нет инструментов `browser_*` | Схемы MCP не загружены | `ToolSearch` → `select:mcp__playwright__browser_navigate,...` |
