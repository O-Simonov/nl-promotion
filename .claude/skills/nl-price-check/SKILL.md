# Проверка цены товара NL по названию

Скилл достаёт актуальную цену, наличие и характеристики любого товара NL International прямо с витрины интернет-магазина через Playwright MCP (живой браузер). Цены динамические и зависят от региона — поэтому берём их с сайта, а не из памяти.

## Когда применять

- «Зайди на сайт NL и проверь цену <товар>»
- «Сколько сейчас стоит Energy Diet / Greenflash / Lab / …»
- Нужно наличие («есть / нет в наличии») и фасовка/артикул
- Нужно собрать цены сразу по нескольким товарам или по всей линейке

## Предусловие

Playwright MCP-сервер уже установлен в проекте (браузер для чтения динамических страниц NL). Если инструменты `mcp__playwright__browser_*` недоступны — сначала загрузить их схемы через ToolSearch (`select:mcp__playwright__browser_navigate,...`).

---

## Ключевые факты о сайте (проверено)

1. **Магазин — это SPA на `ng.nlstar.com`**, НЕ `nlstar.com`.
   `nlstar.com` — корпоративный сайт, его `/search/` отдаёт **404**.
2. **Рабочий поиск:** `https://ng.nlstar.com/ru/search?q=<запрос>`
   (параметр именно `q`; при ручном вводе в поле поиска URL сам станет `?q=...`).
3. **Контент грузится асинхронно.** Сразу после `navigate` блок `main` пустой — обязательно `browser_wait_for` 2–3 секунды, затем `browser_snapshot`.
4. **Цена и наличие зависят от региона** (по умолчанию Москва). Для другого города — сменить регион кнопкой «Москва | ru» в шапке.
5. **Единая цена за вкус.** У линеек вроде ED Smart все классические вкусы стоят одинаково; «сеты» (наборы) стоят кратно дороже — не путать сет с одиночным товаром.

## Процедура

### Шаг 1 — Открыть поиск магазина
```
mcp__playwright__browser_navigate  url: https://ng.nlstar.com/ru/search?q=<URL-encoded запрос>
```
Пробел кодировать как `%20` или `+`. Пример: `?q=Energy%20Diet`.

### Шаг 2 — Дождаться отрисовки и снять снапшот
```
mcp__playwright__browser_wait_for   time: 3
mcp__playwright__browser_snapshot
```
Если в шапке всплыл cookie-баннер с кнопкой «OK» — он не мешает чтению, можно игнорировать. Если мешает клику — нажать кнопку «OK».

> Запасной путь, если страница поиска не отрисовала результаты: ввести запрос в само поле поиска —
> `browser_type` в `searchbox "Искать в NL Store…"` с `submit: true`, потом снова `wait_for` + `snapshot`.

### Шаг 3 — Найти товар в результатах
В снапшоте раздел `heading "Продукция"` содержит карточки: название (ссылка вида `/url: /ru/product/<id>` или `/ru/set/<slug>`), цена `<сумма> ₽` и статус («В корзину» / «Выбрать» / «Нет в наличии»). Выбрать нужную карточку по названию.

### Шаг 4 — Открыть карточку товара и подтвердить цену
```
mcp__playwright__browser_navigate  url: https://ng.nlstar.com/ru/product/<id>
mcp__playwright__browser_wait_for  time: 2
mcp__playwright__browser_snapshot      (или browser_take_screenshot для наглядности)
```
На карточке надёжно читаются: точное название, **цена** (первое `<сумма> ₽` — это цена товара; остальные суммы ниже относятся к блокам «рекомендуем/часто покупают»), артикул, фасовка («N порций по M г»), статус наличия, КБЖУ.

### Шаг 5 — Закрыть браузер
```
mcp__playwright__browser_close
```

---

## Чтение цены из большого снапшота

Снапшот карточки крупный и сохраняется в файл. Чтобы вытащить только цены, не читая весь файл:
```
Grep  pattern: \d[\d  ]*₽   output_mode: content   -o: true   path: <путь к сохранённому снапшоту>
```
**Первая** найденная сумма — цена самого товара.

## Формат ответа пользователю

```
Актуальная цена (NL Store, регион <город>, <дата>):

Товар:     <точное название>
Цена:      <сумма> ₽
Фасовка:   <N порций по M г / объём>
Артикул:   <id>
Наличие:   в наличии / нет в наличии
```
Если запрос дал несколько товаров (вкусы, объёмы, сеты) — дать короткую таблицу и пометить, что цена единая за вкус, а сеты считаются отдельно.

## Диагностика

| Симптом | Причина | Решение |
|---------|---------|---------|
| 404 / «Что-то пошло не так» | Открыт `nlstar.com` вместо магазина | Использовать `ng.nlstar.com` |
| `main` пустой в снапшоте | SPA не успела отрисоваться | `browser_wait_for` 3 сек и снять снапшот заново |
| «Target page … has been closed» | Гонка между navigate и snapshot | Повторить `navigate`, затем `wait_for`, затем `snapshot` |
| Цена не та, что ожидал | Другой регион / выбран «сет», а не товар | Сменить регион; открыть именно `/ru/product/<id>` |
| Нет инструментов `browser_*` | Схемы MCP не загружены | `ToolSearch` → `select:mcp__playwright__browser_navigate,...` |

## Служебные файлы

Playwright MCP складывает снапшоты/логи в `.playwright-mcp\`, а скриншоты — в корень (`*.jpeg`/`*.png`). Это рабочий мусор — не коммитить. При необходимости добавить в `.gitignore`:
```
.playwright-mcp/
*.playwright-screenshot.*
```
