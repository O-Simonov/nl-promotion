# X-бот (Twitter) — браузерная публикация через Playwright

В отличие от Telegram/Facebook/Instagram/ВК (официальный API), X-бот публикует
**через браузерную автоматизацию (Playwright)**, логинясь логин/паролем аккаунта.

Причина: API X для новых разработчиков перестал быть бесплатным (с февраля 2026 —
pay-per-use, $0.20 за пост со ссылкой). Браузерный способ бесплатен, но требует
установки Node.js + Playwright и **один ручной вход** (см. ниже).

## Как это работает

- `Post-Next-X.ps1` — оркестратор: берёт следующий пост из общей очереди
  `tg-bot\queue\`, обрезает подпись до 280 символов **с сохранением реф-ссылки NL**,
  вызывает Node-воркер, при успехе переносит файлы в `x-bot\sent\`, пишет лог,
  повторяет 4 раза, после 3 сбоев подряд шлёт уведомление владельцу.
- `post-x.mjs` — Playwright-воркер: логинится (или переиспользует сессию),
  открывает форму твита, вводит текст, прикрепляет картинку, жмёт «Post».
- `Test-Bot-X.ps1` — проверяет вход без публикации.
- `Setup-Schedule-X.ps1` — создаёт задачу `NL-X-AutoPost` на 13:07 ежедневно.

## Установка (один раз)

### 1. Установить Node.js 20+ (если нет)
```powershell
winget install OpenJS.NodeJS.LTS
# перезапусти терминал, проверь:
node --version
```

### 2. Поставить Playwright + браузер
```powershell
cd C:\NL_produkt\x-bot
npm install playwright
npx playwright install chromium
```
(Браузер ~150 МБ скачается в `%LocalAppData%\ms-playwright\`.)

### 3. Конфиг
```powershell
Copy-Item config.example.json config.json
notepad config.json   # впиши login и password аккаунта X
```
`config.json` в `.gitignore` — не попадёт в git. Поля:
```json
{ "login": "olegsimnov", "password": "***", "headless": true }
```
`headless: false` — окно браузера видно (удобно при первом входе и отладке).

## Первый запуск (важно!)

X **троттлит автоматический вход** с серверов/ботов: при попытке залогиниться
скриптом ответит «We've temporarily limited your login. Please try again later.»
Поэтому первый вход делаем **вручную** через `Login-X.ps1` — он открывает видимый
браузер, ты логинишься сам (минуя челлендж/2FA как человек), сессия сохраняется в
`session.json`. Дальше бот ходит сам по сессии, без повторного логина, и челленджи
появляются редко.

```powershell
# 1) Ручной вход (окно видно). Логинься сам в открывшемся браузере;
#    дождись ленты X — скрипт сам поймёт, что вошёл, и сохранит session.json.
pwsh -File C:\NL_produkt\x-bot\Login-X.ps1

# 2) Проверка входа по сохранённой сессии (без публикации):
pwsh -File C:\NL_produkt\x-bot\Test-Bot-X.ps1

# 3) Расписание (теперь headless:true можно оставить — сессия уже есть):
pwsh -File C:\NL_produkt\x-bot\Setup-Schedule-X.ps1
```

Запускай `Login-X.ps1` **в своей интерактивной сессии** (RDP/консоль), чтобы окно
браузера было видно и им можно было управлять. Если X прямо сейчас троттлит вход
даже ручной — подожди несколько часов и повтори (кулдаун временный).

Если сессия потом слетит (бот упадёт с `needManual`) — повтори `Login-X.ps1`.

## Ограничения и нюансы

- **280 символов.** Подпись обрезается; тело сокращается, реф-ссылка `nlstar.com`
  сохраняется, хэштеги отбрасываются (если не помещаются). Факт обрезки пишется в лог.
- **Хрупкость.** Любой редизайн X может сломать селекторы → бот упадёт, владелец
  получит уведомление после 3 сбоев. Чинишь `post-x.mjs` по скриншоту `logs\x-error.png`.
- **session.json** — не коммитить (уже в `.gitignore`).
- **Аккаунт, не страница.** В отличие от FB (страница) X постит от личного аккаунта
  `olegsimnov`. (См. анти-паттерн в методичке — для X это допустимо, т.к. отдельных
  «страниц» нет.)
- Для автоматизированных аккаунтов X требует пометку «bot» в био — имей в виду.
