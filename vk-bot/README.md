# VK-бот для автопостинга в сообщество ВКонтакте.

Работает в паре с Telegram-ботом: оба читают из общей очереди `../tg-bot/queue/`.
TG-бот берёт первый файл → публикует в 10:00 → переносит в `../tg-bot/sent/`.
VK-бот берёт первый файл → публикует в 14:00 → переносит в `sent/`.

## Файлы

- **`Post-Next-VK.ps1`** — публикация одного поста (читает очередь, шлёт в ВК, переносит в sent).
- **`Test-Bot-VK.ps1`** — проверка, что токен рабочий и сообщество доступно.
- **`Setup-Schedule-VK.ps1`** — создаёт задачу в Windows Task Scheduler (ежедневно в 14:00).
- **`config.json`** — токен и ID сообщества (НЕ коммитится).

## Настройка (один раз)

> ⚠️ **Главная засада — токен для КАРТИНОК.**
> Токен СООБЩЕСТВА (из «Управление → Работа с API») умеет постить только текст.
> На загрузке фото он падает: `photos.getWallUploadServer` отдаёт
> **`error 27 — Group authorization failed: method is unavailable with group auth`**.
> Чтобы постить С КАРТИНКАМИ — нужен **ПОЛЬЗОВАТЕЛЬСКИЙ** токен аккаунта,
> который является **админом** сообщества. Бот всё равно публикует от имени
> сообщества (в коде `from_group=1`), но загрузить фото может только user-токен.

### Как получить рабочий токен (с правом на фото) — проверено 2026-06

VK закрыл простые пути: создание своего приложения на dev.vk.com упирается в **VK Бизнес ID**
с требованием ИНН, а готовое приложение «VK Admin» (`client_id=6121396`) отдаёт
`application is blocked`. Сработал **Kate Mobile** через implicit-flow:

1. Открой ссылку в браузере под аккаунтом-**админом** сообщества и нажми «Разрешить»:
   ```
   https://oauth.vk.com/authorize?client_id=2685278&scope=photos,wall,groups,offline&redirect_uri=https://oauth.vk.com/blank.html&display=page&response_type=token&revoke=1&v=5.199
   ```
2. Браузер откроет почти пустую страницу `oauth.vk.com/blank.html`, а в АДРЕСНОЙ СТРОКЕ будет:
   `...#access_token=vk1.a.XXXXXXXX&expires_in=0&user_id=...`
   (`expires_in=0` = токен бессрочный, это даёт scope `offline`).
3. Скопируй кусок **между `access_token=` и `&expires_in`** — это и есть токен.
4. Впиши его в `config.json` → `botToken`. `groupId` — число без `club`.
5. Проверь: `pwsh -File Test-Bot-VK.ps1` — увидишь имя сообщества.
   Полная проверка фото: `pwsh -File Post-Next-VK.ps1` — пост должен выйти С КАРТИНКОЙ.
6. Расписание: `pwsh -File Setup-Schedule-VK.ps1` — задача ежедневно в 14:00.

### Запасной вариант — посты без картинок
Если token-путь VK прикроет совсем, токеном СООБЩЕСТВА можно постить ТОЛЬКО ТЕКСТ
(`wall.post` с group-токеном работает). Тогда из `Post-Next-VK.ps1` нужно убрать блок
загрузки фото (`photos.getWallUploadServer` / `photos.saveWallPhoto`).

## Важно

- **Скопируй** `config.example.json` в `config.json` и впиши токен (см. выше — нужен USER-токен).
- Никогда не пиши токен в чат/коммиты — `config.json` в `.gitignore`.
- Если токен утечёт — отзови доступ приложения Kate Mobile в настройках безопасности ВК
  (или смени пароль) и получи новый токен по ссылке выше.
- Расписание работает **только когда ПК включён и пользователь залогинен**
  (задача запускается в режиме Interactive; StartWhenAvailable добьёт пропуск при следующем включении).
